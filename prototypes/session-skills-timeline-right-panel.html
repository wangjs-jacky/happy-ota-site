Fallback when no trace exists

If the session has no usable skill activation evidence yet, the panel can fall back to the existing installed-skills preview instead of staying empty.

Why this works

The top card answers the immediate question fast: which skills are shaping the turn now. The timeline underneath explains order and purpose without needing a wide layout.

Key distinction

  • Installed skills are machine-level inventory.
  • Used skills are session-level behavior trace.
  • The right panel should default to the second, not the first.

Data model hint

Prototype assumes two confidence levels: confirmed from explicit events and system/inferred from reconstruction. That lets the UI ship before the instrumentation is perfect.
